# LLM-Optimized Documentation

When implementing, debugging, or making decisions about a technology, check the documentation first. Many libraries publish `llms.txt` or `llms-full.txt` files designed for LLM consumption. Prefer these over relying on training data.

## How to Use

1. If the technology has an `llms.txt` link, fetch it to get a structured index of available docs.
2. If it has an `llms-full.txt` link, fetch it to get the complete documentation in a single file.
3. If it only has a docs link, fetch the most relevant page for your task.
4. Use these docs to verify APIs, patterns, and best practices before writing code.

## When to Fetch Docs

- Before using an API you haven't verified in the current session
- When an error suggests a breaking change or deprecated method
- When choosing between multiple patterns for a library
- When the user asks about a technology listed above
